01 / Introduction

What is a Frame Buffer?

A frame buffer is a dedicated memory area that stores the color value of every pixel that appears on your screen.

FRAME_BUFFER_SIMULATION
Resolution 16 x 16 pixels
Memory Used 768 bytes
Active Pixels 0

Definition

The frame buffer acts as a digital canvas where each memory location corresponds to a single pixel on the display screen.

Pixel(x, y) = Memory[width * y + x]
Pixel to memory mapping
  • 1
    Storage: Each pixel's color is stored as numerical values in memory
  • 2
    Addressing: Pixel position (x, y) maps to a specific memory address
  • !
    Refresh: Display hardware reads this memory 60+ times per second
02 / Color Model

Pixel Color Representation

Learn how colors are represented using the RGB color model, combining Red, Green, and Blue light intensities.

RGB Color Model

Each pixel stores three values representing the intensity of red, green, and blue light. Values range from 0 (off) to 255 (full intensity).

Memory Layout

The frame buffer is organized as a 2D array where each cell contains the RGB values for one pixel. Memory is typically laid out row by row.

Address = Base + (y * width + x) * bytes_per_pixel
Memory address calculation
  • 1
    Row-Major Order: Pixels stored left-to-right, top-to-bottom
  • 2
    Stride: Number of bytes per row (width * bytes_per_pixel)
  • 3
    24-bit Color: 3 bytes per pixel (R, G, B each 8 bits)
  • 4
    32-bit Color: 4 bytes (R, G, B, Alpha for transparency)

Image Formation Process

Scan-Line Rendering

The display controller reads the frame buffer line by line, sending pixel values to the screen in a process called scan-line rendering.

Scan-line algorithm
  • 1

  • 2
    Double Buffering: Draw to back buffer while front displays
  • 3

How Algorithms Use Frame Buffer

Graphics algorithms determine which pixels need to change and write new color values directly into the frame buffer.
