What is a Line?

In computer graphics, a line is the shortest path between two points in a 2D or 3D space. It's defined by two endpoints P1(x1, y1) and P2(x2, y2).

Pixel Representation

On a computer screen, lines are drawn using pixels. Each pixel is a small square that can be illuminated to create the visual representation of a line.

Mathematical Foundation

Lines follow the equation y = mx + c, where m is the slope and c is the y-intercept. This forms the basis for all line drawing algorithms.

Slope Formula

Calculate slope from two points

m = ( y2 - y1 ) / ( x2 - x1 ) 
// Calculate slope from two points
function calculateSlope(p1, p2) {
    const dx = p2.x - p1.x;
    const dy = p2.y - p1.y;
    
    if (dx === 0) {
        return undefined; // Vertical line
    }
    
    return dy / dx;
}

Types of Lines

The slope of a line determines its orientation. Different slope values create different types of lines.

Horizontal Line

m = 0

A horizontal line has zero slope. All points on the line have the same y-coordinate.

y = c (constant)

Vertical Line

m = undefined

A vertical line has undefined slope. All points have the same x-coordinate. Division by zero occurs.

x = c (constant)

Positive Slope

m > 0

A line with positive slope rises from left to right. As x increases, y also increases.

y = mx + c where m > 0

Negative Slope

m < 0

A line with negative slope falls from left to right. As x increases, y decreases.

y = mx + c where m < 0

Pixel Approximation Concept

Understanding how ideal mathematical lines become pixel-based representations on screen.

The Challenge

A mathematical line is continuous and infinite. However, computer screens are made of discrete pixels. We need to determine which pixels to illuminate to best represent the line.

Rounding to Pixels

When we calculate y = mx + c, we get decimal values. Since pixels are at integer coordinates, we must round these values. This rounding creates the "staircase" effect seen in pixel lines.

The Problem

Calculating y = mx + c for every pixel position requires floating-point multiplication and rounding, which is computationally expensive. This is why we need efficient algorithms.

Transition to Algorithms

Direct calculation using the line equation is inefficient. Specialized algorithms solve this problem.

Why Not Just Use y = mx + c?

1

Floating Point Operations

Multiplication and division are expensive operations. For each pixel, we need to compute y = mx + c.

2

Rounding Errors

Converting floating-point results to integers introduces cumulative rounding errors that affect line quality.

3

Performance

For real-time graphics with millions of lines, we need algorithms that use only integer arithmetic.

DDA

Digital Differential Analyzer

Incremental Algorithm

DDA uses incremental calculations. Instead of computing y = mx + c for each x, it adds the slope incrementally: y(n+1) = y(n) + m 

// DDA approach
let y = y1;
for (let x = x1; x <= x2; x++) {
    plot(x, Math.round(y));
    y = y + slope; // Incremental
}
Pros: Simple to implement Cons: Still uses floating-point
BA

Bresenham Line Algorithm

Integer-Only Algorithm

Bresenham's algorithm uses only integer addition and subtraction. It uses a decision parameter to determine which pixel is closer to the ideal line. 

// Bresenham approach
let d = 2*dy - dx; // Decision param
for (let x = x1; x <= x2; x++) {
    plot(x, y);
    if (d > 0) { y++; d -= 2*dx; }
    d += 2*dy; // Integer only!
}
Pros: Fast, integer-only Note: Industry standard
