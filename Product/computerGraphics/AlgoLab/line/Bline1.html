01 / Introduction

What is Bresenham's Algorithm?

The Gold Standard for Line Drawing

Bresenham's line algorithm, developed by Jack Bresenham in 1962, is the most efficient algorithm for drawing straight lines on raster displays. It uses only integer arithmetic to determine which pixels to illuminate.

Unlike DDA which uses floating-point calculations, Bresenham's algorithm relies on a decision parameter to choose between two candidate pixels at each step, making it significantly faster.

Key Insight: Integer Decision Making
No floating-point operations required

Why is it Faster Than DDA?

The algorithm's speed comes from eliminating expensive operations:

  • 01 No floating-point arithmetic
  • 02 No division or multiplication in loop
  • 03 Only addition, subtraction, and comparison
  • 04 Ideal for hardware implementation

Bresenham is the standard algorithm used in graphics hardware (GPUs) for line drawing because of its efficiency.

02 / Mathematics

Decision Parameter Concept

Initial Calculations

First, compute the differences between endpoints:

dx = x2 - x1
dy = y2 - y1

Initial Decision Parameter

The starting value of the decision parameter:

p0 = 2dy - dx
Initial decision parameter

The decision parameter (p) determines whether the next pixel should be East (same row) or North-East (next row) of the current pixel.

Decision Rules

p<0

Choose East Pixel

Move to (x+1, y) - same row, next column

Update: p = p + 2dy

p≥0

Choose North-East Pixel

Move to (x+1, y+1) - next row, next column

Update: p = p + 2dy - 2dx

Note: The decision parameter essentially measures the distance from the ideal line to the midpoint between the two candidate pixels. A negative value means the line passes below the midpoint (choose East), while a positive value means it passes above (choose North-East).

Bresenham Pseudocode

The complete Bresenham algorithm implementation in pseudocode form. This version assumes slope less than 1 (|dx| > |dy|).

Bresenham_Line_Algorithm

2// Assumption: |dx| > |dy| (slope < 1)
3
4function Bresenham_Line(x1, y1, x2, y2):
5 // Calculate differences
6 dx = x2 - x1
7 dy = y2 - y1
8
9 // Initial decision parameter
10 p = 2 * dy - dx
11
12 // Initialize starting position
13 x = x1
14 y = y1
15
16 // Plot each pixel along the line
17 while x <= x2:
18 plot(x, y)
19
20 // Decision making
21 if p < 0:
22 // East pixel chosen
23 p = p + 2 * dy

25 // North-East pixel chosen
26 y = y + 1
27 p = p + 2 * dy - 2 * dx
28
29 x = x + 1
30 end while
31
32end function
05 / Analysis

Advantages and Disadvantages

Advantages

  • Uses only integer arithmetic (no floating-point)
  • Significantly faster than DDA algorithm
  • More efficient for raster display systems
  • Only uses addition and subtraction in loop
  • Standard algorithm for graphics hardware
  • Produces accurate and smooth lines

Disadvantages

  • More complex logic than DDA to understand
  • Basic version works best when slope < 1
  • Requires modification for other octants
  • Decision parameter concept takes time to grasp

Performance Comparison: DDA vs Bresenham

DDA Algorithm

Operations per pixel:
2 floating-point additions
1 rounding operation

Speed: Slower due to FP operations

Bresenham Algorithm

Operations per pixel:
1-2 integer additions
1 comparison

Speed: Much faster, integer only
