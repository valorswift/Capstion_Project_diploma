01 / Introduction

What is a Line in Computer Graphics?

In computer graphics, a line is defined by two endpoints and represents the shortest path between them on a pixel-based display.

COORDINATE_GRID

Definition

A line segment in computer graphics is defined by two distinct points in a coordinate system:

P1(x1, y1) and P2(x2, y2)
Two endpoints define a line
  • 1
    Start Point P1: The beginning coordinate of the line segment
  • 2
    End Point P2: The ending coordinate of the line segment
  • !
    Challenge: Screens are made of discrete pixels, not continuous coordinates

03 / Mathematics

Line Equations

Understanding the mathematical foundation of lines is essential for developing efficient drawing algorithms.

Slope-Intercept Form

The most common way to represent a line mathematically is using the slope-intercept equation:

y = mx + c
Slope-intercept form

Calculating Slope

The slope measures the steepness of the line:

m = (y2 - y1) / (x2 - x1)
Slope formula

Y-Intercept

The point where the line crosses the y-axis:

c = y1 - m * x1
Y-intercept calculation

Note: Using these equations for every pixel is computationally expensive, which is why we need efficient algorithms like DDA and Bresenham.

Ideal Line vs Pixel Representation

Computer screens use discrete pixels, so we must approximate continuous mathematical lines with discrete points.

The Challenge

Mathematical lines are continuous and have infinite resolution. But pixel displays are discrete grids with fixed positions.

  • 1
    Continuous: Math lines have infinite points between any two points
  • 2
    Discrete: Pixels have fixed integer coordinates only
  • 3
    Approximation: We choose the closest pixels to the ideal line

06 / Next Steps

Why We Need Algorithms

Calculating every point using the line equation is inefficient. Specialized algorithms solve this problem with better performance.

The Problem with Direct Calculation

Using y = mx + c for each x position requires:

  • 1
    Floating-point multiplication for each pixel (slow)
  • 2
    Rounding operations to convert to integers
  • 3
    Special cases for vertical and steep lines

The Solution

Incremental algorithms like DDA and Bresenham use previous calculations to determine the next pixel, avoiding expensive multiplication and division.

DDA Line Algorithm

Uses incremental floating-point addition to calculate pixel positions. Simpler to understand but uses floating-point operations.

Digital Differential Analyzer

Bresenham Line Algorithm

Uses only integer arithmetic with a decision parameter. Most efficient and widely used in graphics hardware.

Integer Only
Direct: y = mx + c (multiplication per pixel)
DDA: y = y + slope (addition per pixel)
