01 / Introduction

What is Seed Fill?

The Seed Fill Algorithm fills a connected region of pixels starting from a chosen seed point, expanding outward until reaching a boundary.

The Core Concept

A pixel is chosen as the "seed" inside a region. The algorithm checks if neighboring pixels should be filled, and continues spreading until the entire region is colored or a boundary is reached.

Key Advantages

    Intuitive: Simple concept - start from a point and spread
  • 2
    Flexible: Works with any shape boundary
  • 3
    Practical: Used in paint bucket tools
  • 4
    Adaptable: 4-connected or 8-connected options
02 / Region Filling

Concept of Region Filling

Region filling works on connected pixels, using boundary and fill colors to determine where to stop.

Two Essential Colors

The algorithm distinguishes between two key colors to determine the fill behavior:

Boundary Color: Stops propagation
Fill Color: Applied to valid pixels
Color-based decision

Fill Condition

A pixel is filled only if:

  • 1
    Not Boundary: Pixel is not the boundary color
  • 2
    Not Filled: Pixel hasn't been filled already
  • 3
    Connected: Pixel is connected to the seed region

05 / Algorithm Logic

Step-by-Step Process

Understanding the logical flow of the seed fill algorithm.

1

Select Seed Pixel

Choose a starting pixel inside the region to be filled.

2

Check Pixel Validity

Verify the pixel is not a boundary and not already filled.

3

Fill the Pixel

Set the pixel to the fill color and mark as processed.

4

Add Neighbors to Queue

Push valid neighboring pixels to the processing queue/stack.

5

Repeat Until Complete

Continue processing until the queue is empty (region filled).

Queue-Based: FIFO (Breadth-First) → Radial spread
Stack-Based: LIFO (Depth-First) → Directional spread
Data structure affects fill pattern
06 / Algorithm Pseudocode

Complete Implementation

The complete pseudocode for the Seed Fill Algorithm using a stack-based approach.

PSEUDOCODE
// Seed Fill Algorithm (Stack-based)
function seedFill(seedX, seedY, fillColor, boundaryColor) {
  // Initialize stack with seed pixel
  stack = [(seedX, seedY)]
  while (stack is not empty) {
    // Pop pixel from stack
    (x, y) = stack.pop()
    // Check if pixel is valid for filling
    if (getPixel(x, y) != boundaryColor &&
        getPixel(x, y) != fillColor) {
      // Fill the pixel
      setPixel(x, y, fillColor)
      // Push neighbors (4-connected)
      stack.push((x + 1, y))
      stack.push((x - 1, y))
      stack.push((x, y + 1))
      stack.push((x, y - 1))
    }
  }
}

Implementation Notes

  • 1
    Boundary Check: Essential to prevent infinite loops
  • 2
    Fill Check: Avoid re-processing filled pixels
  • 3
    Stack Size: Can grow large for big regions

Complexity Analysis

Time: O(n) where n = pixels in region
Space: O(n) for stack in worst case

07 / Comparison & Use Cases

Seed Fill vs Boundary Fill

Understanding when to use each filling algorithm.

Seed Fill (Flood Fill)

Fills all connected pixels that match the original color at the seed point.

  • Works without explicit boundary
  • Replaces all connected same-color pixels
  • Paint bucket tool behavior
  • Stops when color changes

Boundary Fill

Fills until reaching pixels of a specific boundary color.

  • Requires defined boundary
  • Stops at boundary color only
  • More precise control
  • Fills any interior color

Help & Guide

What is Seed Fill?

Seed Fill (also known as Flood Fill) is an algorithm that determines the area connected to a given node in a multi-dimensional array. It is used in the "bucket" fill tool of paint programs to fill connected, similarly-colored areas with a different color.

This visualization demonstrates the boundary fill variant, where the algorithm fills until it hits a specific boundary color.

Simulator Controls

Draw Boundary: Click and drag on the grid to draw walls.
Place Seed: Hold Shift and click inside the area to place the starting point.

Running the Algorithm

Start: Initializes the queue with the seed and pops the first item.
Next Step: Processes one pixel from the queue, fills it, and adds valid neighbors.
Auto: Runs the algorithm continuously until the queue is empty.

Watch the Queue Size to see how the algorithm manages memory. It grows as it discovers neighbors and shrinks as it processes them.
