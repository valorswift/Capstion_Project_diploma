01 / Introduction

What is Boundary Fill Algorithm?

The Boundary Fill Algorithm fills a connected region of pixels until reaching a specified boundary color, regardless of the interior's original color.

Boundary Fill Visualization
Algorithm Boundary Fill
Method Stack-based
Propagation 4-connected

The Core Concept

Starting from a seed pixel inside a boundary, the algorithm fills all connected pixels until it encounters the boundary color. Unlike Flood Fill, it doesn't care about the original interior color.

Seed Point → Check Boundary → Fill Interior
Basic flow of boundary fill

Key Advantages

  • 1
    Precise: Fills until exact boundary color is found
  • 2
    Flexible Interior: Works with any interior color pattern
  • 3
    Controlled: Explicit boundary prevents overflow
  • 4
    Reliable: Predictable fill behavior
02 / Region Filling

Concept of Region Filling

Boundary fill uses boundary color detection to determine where to stop filling.

Two Essential Colors

The algorithm distinguishes between two key colors to determine the fill behavior:

Boundary Color: Stops propagation
Fill Color: Applied to interior pixels
Color-based decision making

Fill Condition

A pixel is filled only if:

  • 1
    Not Boundary: Pixel is not the boundary color
  • 2
    Not Filled: Pixel hasn't been filled already
  • 3
    Inside Region: Connected to the seed through interior

03 / Neighbor Connectivity

4-Connected vs 8-Connected Fill

The algorithm can use different neighbor patterns to propagate, affecting which regions get filled.

4-Connected Fill

Only horizontal and vertical neighbors are considered. Safer for thin boundaries.

05 / Algorithm Logic

Step-by-Step Process

Understanding the logical flow of the boundary fill algorithm.

1

Select Seed Pixel

Choose a starting pixel inside the bounded region to be filled.

2

Check Pixel Against Boundary

Verify the pixel is not the boundary color and not already filled.

3

Fill the Pixel

Set the pixel to the fill color and mark as processed.

4

Push Neighbors to Stack

Add valid neighboring pixels (4 or 8 connected) to the processing stack.

5

Repeat Until Complete

Continue processing until the stack is empty (entire region filled).

Stop Condition: pixel == boundary_color OR pixel == fill_color
Continue: pixel != boundary_color AND pixel != fill_color

06 / Algorithm Pseudocode

Complete Implementation

The complete pseudocode for the Boundary Fill Algorithm using a stack-based approach.

PSEUDOCODE
// Boundary Fill Algorithm (Stack-based)
function boundaryFill(seedX, seedY, fillColor, boundaryColor) {
  // Initialize stack with seed pixel
  stack = [(seedX, seedY)]
  while (stack is not empty) {
    // Pop pixel from stack
    (x, y) = stack.pop()
    // Check if pixel should be filled
    // NOT boundary AND NOT already filled
    if (getPixel(x, y) != boundaryColor &&
        getPixel(x, y) != fillColor) {
      // Fill the pixel
      setPixel(x, y, fillColor)
      // Push 4-connected neighbors
      stack.push((x + 1, y))
      stack.push((x - 1, y))
      stack.push((x, y + 1))
      stack.push((x, y - 1))
      // For 8-connected, also add:
      // stack.push((x+1, y+1))
      // stack.push((x+1, y-1))
      // stack.push((x-1, y+1))
      // stack.push((x-1, y-1))
    }
  }
}

Implementation Notes

  • 1
    Boundary Check: Essential to stop at boundary color
  • 2
    Fill Check: Avoid re-processing already filled pixels
  • 3
    Stack Size: Can grow large for big regions

Complexity Analysis

Time: O(n) where n = pixels in region
Space: O(n) for stack in worst case

07 / Comparison & Use Cases

Boundary Fill vs Seed Fill

Understanding when to use each filling algorithm.

Boundary Fill

Fills until reaching a specific boundary color. Works regardless of interior colors.

  • Requires defined boundary color
  • Stops at boundary color only
  • Works with any interior pattern
  • More precise boundary control

Seed Fill (Flood Fill)

Fills all connected pixels matching the seed's original color until color changes.

  • No explicit boundary needed
  • Replaces same-color regions
  • Paint bucket tool behavior
  • Stops when color changes
