01 / Introduction

What is the Midpoint Circle Algorithm?

An efficient algorithm that draws circles by using a midpoint decision parameter to select the pixel closest to the ideal circle boundary.

Circle Rasterization
Ideal Circle Mathematical curve
Rasterized Discrete pixels
Method Midpoint Decision

The Core Idea

Instead of calculating the circle equation for every pixel, the midpoint algorithm uses a decision parameter that evaluates the position of a midpoint between two candidate pixels relative to the circle boundary.

Midpoint Test
Which pixel is closer to the circle?

Key Advantages

  • 1
    Integer Only: All calculations use integer arithmetic
  • 2
    Incremental: Decision parameter updated incrementally
  • 3
    Efficient: Only one decision per pixel
  • 4
    Accurate: Produces visually correct circles
02 / Circle Equation

Mathematical Foundation

Understanding the circle equation is essential for deriving the midpoint decision parameter.

Circle Definition

A circle is defined as the set of all points at a fixed distance (radius) from a center point.

(x - xc)² + (y - yc)² = r²
Circle equation with center (xc, yc) and radius r

Key Components

  • C
    Center (xc, yc): The fixed center point of the circle
  • r
    Radius r: Distance from center to any point on circle
  • P
    Point (x, y): Any point satisfying the equation lies on the circle
f(x, y) = (x - xc)² + (y - yc)² - r²

f(x, y) < 0 → inside circle
f(x, y) = 0 → on circle
f(x, y) > 0 → outside circle

04 / Circle Symmetry

8-Way Symmetry

Circles exhibit perfect 8-way symmetry, allowing computation in only one octant.

Symmetry Principle

07 / Algorithm Pseudocode

Complete Implementation

The complete pseudocode for the Midpoint Circle Algorithm using integer arithmetic.

PSEUDOCODE
// Midpoint Circle Drawing Algorithm
function midpointCircle(xc, yc, r) {
  // Initialize starting point and decision parameter
  x = 0
  y = r
  p = 1 - r // Initial decision parameter
  // Loop until we've drawn one octant
  while (x <= y) {
    // Plot all 8 symmetric points
    drawPixel(xc + x, yc + y)
    drawPixel(xc + y, yc + x)
    drawPixel(xc - y, yc + x)
    drawPixel(xc - x, yc + y)
    drawPixel(xc - x, yc - y)
    drawPixel(xc - y, yc - x)
    drawPixel(xc + y, yc - x)
    drawPixel(xc + x, yc - y)
    // Update decision parameter and position
    if (p < 0) {
      // Midpoint inside circle - choose East
      p = p + 2 * x + 3
    } else {
      // Midpoint outside/on circle - choose South-East
      p = p + 2 * (x - y) + 5
      y = y - 1
    }
    x = x + 1
  }
}

Key Observations

  • 1
    Initial p: Starts at 1 - r (not 3 - 2r like Bresenham)
  • 2
    Update Formula: Uses 2x+3 and 2(x-y)+5 (simpler coefficients)
  • 3
    Symmetry: All 8 points plotted per iteration

Complexity Analysis

Time: O(r)
Space: O(1)
Algorithm complexity

The algorithm runs in linear time relative to radius, with constant memory usage. Only r/√2 iterations needed for the complete circle.
