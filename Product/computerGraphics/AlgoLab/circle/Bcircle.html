What is Bresenham Circle Algorithm?

An efficient algorithm that draws circles using only integer arithmetic, avoiding expensive floating-point calculations while maintaining pixel-perfect accuracy.

The Core Goal

The Bresenham Circle Algorithm selects the pixel closest to the ideal circle path using only integer addition and subtraction. This makes it incredibly fast and hardware-friendly.

03 / Decision Parameter

The Decision Parameter Concept

The algorithm uses a decision parameter to choose between two candidate pixels at each step.

Initial Decision Parameter

The decision parameter is initialized based on the circle radius and updated incrementally as pixels are plotted.

p₀ = 3 - 2r
Initial decision parameter

Two Possible Cases

At each step, we choose between two candidate pixels based on the decision parameter value:

p < 0

Choose pixel (x+1, y)

p = p + 4x + 6

p ≥ 0

Choose pixel (x+1, y-1)

p = p + 4(x-y) + 10

The decision parameter essentially measures which candidate pixel is closer to the ideal circle. Only simple integer additions are needed.

06 / Algorithm Pseudocode

Complete Algorithm Implementation

The complete pseudocode for the Bresenham Circle Algorithm using integer arithmetic.

PSEUDOCODE
// Bresenham Circle Drawing Algorithm
function bresenhamCircle(xc, yc, r) {
  // Initialize starting point and decision parameter
  x = 0
  y = r
  p = 3 - 2 * r // Initial decision parameter
  // Loop until we've drawn one octant
  while (x <= y) {
    // Plot all 8 symmetric points
    drawPixel(xc + x, yc + y)
    drawPixel(xc + y, yc + x)
    drawPixel(xc - y, yc + x)
    drawPixel(xc - x, yc + y)
    drawPixel(xc - x, yc - y)
    drawPixel(xc - y, yc - x)
    drawPixel(xc + y, yc - x)
    drawPixel(xc + x, yc - y)
    // Update decision parameter and position
    if (p < 0) {
      p = p + 4 * x + 6
    } else {
      p = p + 4 * (x - y) + 10
      y = y - 1
    }
    x = x + 1
  }
}

Key Observations

  • 1
    Loop Condition: x ≤ y ensures we stay in the first octant
  • 2
    Symmetry: All 8 points plotted per iteration
  • 3
    Incremental: Only additions, no multiplication per pixel

Complexity Analysis

Time: O(r)
Space: O(1)
Algorithm complexity

The algorithm runs in linear time relative to the radius, with constant memory usage. Only r/√2 iterations are needed for the complete circle.
