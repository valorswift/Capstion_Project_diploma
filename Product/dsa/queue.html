What is a Queue?

A linear data structure that follows a specific order in which operations are performed.

FIFO Principle

The order is First-In-First-Out. The first element added is the first one to be removed.

Real-Life Analogy

A line of people waiting for a bus ticket. The person at the front gets served first. New people join at the rear.

Circular Queue

Solving the problem of memory wastage in simple queues.

The Concept

In a simple queue, once `rear` reaches the end, we can't add more items even if the front is empty. A circular queue connects the end back to the start.

Applications

  • Keyboard Buffer: Strokes are stored in a ring buffer.
  • Traffic Systems: Traffic light cycles.
  • CPU Scheduling: Round Robin algorithm.

Priority Queue

Elements are served based on priority, not arrival time.

How it Works

Each element has a priority number. Higher priority (usually a larger number or lower number depending on impl) is dequeued first.

Task A P:10
Task B P:5
Task C P:1

Applications

  • CPU Scheduling: High priority processes get CPU time first.
  • Dijkstra's Algorithm: Selecting the next shortest path node.
  • Interrupt Handling: Hardware interrupts have strict priorities.

Double Ended Queue (Deque)

Insertion and deletion can happen from both front and rear.

Operations

Input Restricted Deque

Insertion at one end, deletion at both.

Output Restricted Deque

Deletion at one end, insertion at both.

Applications

Used in Sliding Window problems, Undo/Redo stacks (conceptually), and Palindrome Checkers.

Queues in Operating Systems

The OS is essentially a massive queue manager.

Process Scheduling

  • Ready Queue: Processes waiting for CPU time.
  • Device Queue: Processes waiting for I/O (disk, network).
  • Job Queue: All processes in the system.
