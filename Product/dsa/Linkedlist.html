What is a Linked List?

A linear data structure where elements are not stored in contiguous memory locations.

Anatomy of a Node

Each element in a linked list is called a Node. A node contains two parts:

  • Data: The value to be stored.
  • Pointer (Next): The memory address of the next node.
[ Data | Next ]
[ Data | NULL ]

Linked List in Memory

Understanding non-contiguous memory allocation in the Heap.

RAM Representation

Unlike arrays, linked list nodes can be scattered anywhere in the heap. They are connected via pointers (addresses).

0x1004
Data: 10
Next: 0x2048
0x2048
Data: 25
Next: 0x3050
0x3050
Data: 40
Next: NULL
  • Heap Allocation: Nodes are created dynamically using malloc (C) or new (Java/JS).
  • Pointer Dereferencing: CPU follows the address stored in "Next" to find the next instruction.
  • Fragmentation: Over time, free memory becomes scattered, making it hard to find large contiguous blocks (unlike arrays).

Singly vs Doubly Linked List

FeatureSingly Linked ListDoubly Linked List
Pointers One (Next) Two (Prev, Next)
Traversal Forward only Forward & Backward
Memory Less overhead More overhead (extra pointer)
Deletion Requires prev node pointer Easier (can go backwards)

Time Complexity

OperationTime ComplexityExplanation
Insert Head O(1) Just update Head pointer. No traversal needed.
Insert Tail O(1)* O(1) if we maintain a Tail pointer. O(n) otherwise.
Insert At Index O(n) Must traverse from head to the index.
Search O(n) Must visit every node sequentially.
Delete O(n) Must find the node before the one to delete.
