Understanding Hierarchical Clustering

Hierarchical Clustering is an unsupervised learning algorithm that builds a hierarchy of clusters by progressively merging or dividing groups. Unlike K-Means, it doesn't require specifying the number of clusters beforehand.

There are two approaches: Agglomerative (bottom-up) starts with each point as its own cluster and merges the closest pairs, while Divisive (top-down) starts with all points in one cluster and recursively splits them.

  • Linkage Methods Ways to measure distance between clusters: single, complete, average, or Ward's method.
  • Distance Metrics Euclidean, Manhattan, or other metrics for measuring point-to-point distances.
  • Merge Levels Each iteration merges the two closest clusters, building the hierarchy level by level.

Hierarchical Clustering Guide

What is Hierarchical Clustering?

Hierarchical Clustering is an unsupervised learning algorithm that builds a hierarchy of clusters by iteratively merging or dividing groups of data points. It creates a tree-like structure called a dendrogram that shows the relationships between clusters.

What is a Dendrogram?

A dendrogram is a tree diagram that shows the hierarchical relationship between clusters. The vertical axis represents the distance or dissimilarity at which clusters merge. By cutting the dendrogram at different heights, you can obtain different numbers of clusters.

How Cluster Merging Works

  • Single Linkage: Distance between clusters = minimum distance between any two points.
  • Complete Linkage: Distance between clusters = maximum distance between any two points.
  • Average Linkage: Distance between clusters = average distance between all pairs of points.

Hierarchical vs K-Means

Unlike K-Means, Hierarchical Clustering doesn't require specifying K in advance. It produces a hierarchy that allows analysis at multiple levels. K-Means is faster for large datasets but Hierarchical Clustering provides more interpretability and works better with non-spherical clusters.
